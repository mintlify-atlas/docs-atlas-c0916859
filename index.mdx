Prometheus Node Exporter collects system-level metrics from Linux and other Unix-like operating systems and exposes them in the Prometheus exposition format. It runs as a lightweight daemon on each host you want to monitor, serving metrics on port 9100 by default.

## What node_exporter collects

Node exporter exposes metrics about the underlying host, not the container it runs in. It reads from `/proc`, `/sys`, and other kernel interfaces to provide metrics covering:

- **CPU** ‚Äî usage, frequency, per-core stats, vulnerability info
- **Memory** ‚Äî utilization, swap, NUMA topology
- **Disk** ‚Äî I/O stats, filesystem usage, NVMe and hardware monitoring
- **Network** ‚Äî interface stats, conntrack, netstat, network routes
- **System** ‚Äî load average, uptime, systemd unit status, entropy

## How it works

Node exporter runs a single HTTP server. Prometheus scrapes the `/metrics` endpoint at a configurable interval to collect all enabled collectors' output.
